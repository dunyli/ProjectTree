Данный проект представляет собой реализацию двух взаимосвязанных структур данных на языке программирования C: 
красно-чёрного самобалансирующегося двоичного дерева поиска и ассоциативного массива (словаря), построенного поверх этого дерева. 
Проект включает полный набор операций для обеих структур, а также обширную систему тестов и примеров практического применения. 
Все операции имеют логарифмическую сложность O(log n), что обеспечивает высокую производительность даже при работе с большими объёмами данных.

Красно-чёрное дерево — это самобалансирующееся двоичное дерево поиска, 
которое гарантирует логарифмическую сложность O(log n) для операций вставки, удаления и поиска. 
Балансировка достигается за счёт соблюдения пяти строгих свойств:

1) Каждый узел окрашен в красный или чёрный цвет
2) Корень дерева всегда чёрный
3) Все NULL-листья считаются чёрными
4) У красного узла оба потомка должны быть чёрными (запрещены два красных подряд)
5) Для каждого узла все пути до листьев содержат одинаковое количество чёрных узлов

Реализованные операции

Создание и удаление:

rb_create() — создание пустого дерева с инициализацией корня NULL

rb_destroy() — рекурсивное удаление всех узлов и освобождение памяти

rb_create_node() — создание нового узла, копирование ключа

Вставка:

rb_insert() — вставка пары ключ-значение с обновлением существующего ключа

rb_insert_fixup() — восстановление свойств после вставки (3 случая)

Поиск:

rb_search() — поиск узла по ключу с обходом дерева

rb_contains() — проверка наличия ключа

rb_get() — получение значения по ключу

Удаление:

rb_delete() — удаление ключа с тремя случаями (нет левого, нет правого, оба есть)

rb_delete_fixup() — восстановление свойств после удаления (4 случая)

rb_transplant() — замена одного поддерева другим

rb_minimum() — поиск минимального узла в поддереве

Вспомогательные:

rb_rotate_left() — левый поворот вокруг узла

rb_rotate_right() — правый поворот вокруг узла

rb_inorder() — обход дерева в прямом порядке с callback

rb_print() — печать дерева для отладки


Ассоциативный массив (словарь, map) — это структура данных, хранящая пары "ключ-значение" с уникальными ключами. 
В данной реализации словарь является обёрткой над красно-чёрным деревом, предоставляющей удобный интерфейс для работы с данными. 
Все операции наследуют логарифмическую сложность дерева.

Реализованные операции

Создание и удаление:

map_create() — создание словаря с внутренним деревом

map_destroy() — удаление словаря и освобождение памяти

Основные операции:

map_put() — вставка пары ключ-значение, обновление существующего

map_get() — получение значения по ключу

map_contains() — проверка наличия ключа

map_remove() — удаление ключа

Вспомогательные:

map_size() — получение количества элементов

map_is_empty() — проверка на пустоту

map_clear() — очистка словаря

map_for_each() — итерация по всем элементам с callback

map_print() — печать содержимого словаря


Тесты

Группа 1: Тесты красно-чёрного дерева (4 теста)

test_rb_create_destroy — создание и удаление

- Проверка корректного создания пустого дерева
- Проверка инициализации полей (root = NULL, size = 0)
- Проверка освобождения памяти без утечек

test_rb_insert_search — вставка и поиск

- Вставка 5 элементов и проверка размера
- Поиск каждого вставленного элемента
- Поиск отсутствующих ключей
- Обновление существующего ключа (размер не меняется)

test_rb_delete — удаление

- Вставка 7 элементов
- Удаление из разных позиций (лист, середина, корень)
- Проверка правильности обновления указателей
- Повторное удаление и удаление несуществующего ключа

test_rb_edge_cases — краевые случаи

- Работа с пустым деревом
- Дерево из одного элемента
- Удаление единственного элемента

Группа 2: Тесты ассоциативного массива (5 тестов)

test_map_basic — создание и базовые операции

- Создание словаря
- Проверка размера и пустоты
- Корректное удаление

test_map_put_get — вставка и поиск

- Вставка 5 элементов
- Поиск значений
- Проверка отсутствующих ключей
- Работа с NULL-указателем для значения

test_map_update — обновление значений

- Однократное и многократное обновление
- Проверка неизменности размера
- Корректность итогового значения

test_map_remove — удаление

- Удаление нескольких элементов
- Проверка отсутствия удалённых
- Сохранность оставшихся элементов
- Удаление несуществующего ключа

test_map_clear — очистка словаря

- Вставка 10 элементов
- Полная очистка
- Проверка пустоты и отсутствия элементов
- Добавление после очистки
